שלב היישום (Software Deployment)

1. תיאור והסבר כיצד היישום משתמש במודל

היישום הוא אפליקציית ווב (Gradio) המאפשרת למשתמש להעלות הקלטת קול לב (WAV) ולקבל גרף צבעוני של סגמנטציה — סיווג כל נקודת זמן ל-S1, Systole, S2, Diastole או Unrecognized.

תהליך העבודה:

  1. העלאת קובץ WAV — המשתמש מעלה הקלטת PCG דרך הממשק. מיד מוצג גרף אמפליטודה של הסיגנל.
  2. עיבוד מקדים — נורמליזציה, סינון פס-רצועה (60–150 Hz) ו-Downsampling ל-1,000 Hz.
  3. חיתוך לחלונות של 2 שניות — בהתאם לגודל הקלט שעליו אומן המודל (2,000 נקודות). שארית קצרה מ-2s מטופלת בחלון חופף.
  4. חיזוי — כל חלון מוזן למודל (model.predict), שמחזיר הסתברות ל-5 מחלקות בכל נקודת זמן. argmax בוחר את המחלקה.
  5. הצגת תוצאות — גרף Plotly אינטראקטיבי (זום + גלילה) עם צביעה לפי הסגמנטציה.

המודל נטען פעם אחת בעת הפעלת השרת (keras.models.load_model) ונשאר בזיכרון.

2. תיאור קוד הקולט את ה-DATA שעליו יבוצע החיזוי והתאמתו למבנה נתונים המתאים לחיזוי

2.1 עיבוד מקדים — preprocess_wav(audio_path)

קריאת קובץ WAV → המרה למונו (אם סטריאו) → נורמליזציה לטווח [-1,1] לפי max(|signal|) → סינון Butterworth פס-רצועה (60–150 Hz) להסרת רעשים → Downsampling לקצב 1,000 Hz (בהתאם לאימון).

2.2 חיתוך לחלונות — seperate_signal_into_2s_chunks(signal, sr)

הסיגנל נחתך לחלונות בני 2 שניות (2,000 נקודות). אם נשאר "זנב" קצר מ-2s, נוצר חלון חופף שמתחיל 2s לפני סוף הסיגנל — באזור החפיפה נשמר החיזוי מהחלון המלא הקודם.

2.3 התאמה למבנה הנתונים של המודל

כל חלון עובר reshape ל-(1, 2000, 1) — Batch=1, ציר זמן=2000, ערוץ יחיד — בהתאם לקלט שעליו אומן המודל (Input(shape=(2000, 1))). הפלט (1, 2000, 5) מכיל הסתברות ל-5 מחלקות בכל נקודת זמן, ו-argmax בוחר את המחלקה הסבירה ביותר.

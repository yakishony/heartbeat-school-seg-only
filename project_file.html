שלב בניית ואימון המודל (Build and Train Deep Learning Model)

1. תיאור גרפי של ארכיטקטורת המודל

המודל הוא רשת עצבית עמוקה מסוג Conv1D-BiGRU לסגמנטציה בלבד, המבוססת על ארכיטקטורת U-Net. הרשת מקבלת כקלט הקלטת PCG בת 2 שניות (2,000 נקודות זמן לאחר Downsampling) ומפיקה פלט יחיד:

  • ראש סגמנטציה (Segmentation Head) — סיווג כל נקודת זמן לאחד מ-5 מצבים: S1, S2, Systole, Diastole, Background.
Input: (2000, 1) │ ▼ ┌──────────────────────────────────────────────────┐ │ ENCODER │ │ │ │ Conv1D(64, kernel=7) + BatchNorm + Dropout(0.1) │ │ → skip₁ (2000, 64) │ │ MaxPool1D(4) → (500, 64) │ │ │ │ Conv1D(128, kernel=5) + BatchNorm + Dropout(0.2)│ │ → skip₂ (500, 128) │ │ MaxPool1D(4) → (125, 128) │ └──────────────────────┬───────────────────────────┘ │ ▼ ┌──────────────────────────────────────────────────┐ │ BOTTLENECK: Two Stacked BiGRUs │ │ │ │ Bidirectional GRU(128, return_sequences=True) │ │ dropout=0.2, recurrent_dropout=0.2 │ │ → (125, 256) │ │ │ │ Bidirectional GRU(128, return_sequences=True) │ │ dropout=0.2, recurrent_dropout=0.2 │ │ → (125, 256) │ └──────────────────────┬───────────────────────────┘ │ ▼ ┌──────────────────────────────────────────────────┐ │ DECODER (U-Net style) │ │ │ │ UpSampling1D(4) → (500, 256) │ │ Concatenate(skip₂) → (500, 384) │ │ Conv1D(64, 5) + BatchNorm + Dropout(0.2) │ │ │ │ UpSampling1D(4) → (2000, 64) │ │ Concatenate(skip₁) → (2000, 128) │ │ │ │ Dense(5, softmax) → (2000, 5) │ │ ↓ │ │ Per-timestep class: S1/S2/Sys/Dia/BG │ └──────────────────────────────────────────────────┘

כפי שניתן לראות, ה-Encoder מעבד את הסיגנל ומפיק ייצוג קומפקטי. ב-Bottleneck, שתי שכבות BiGRU מוערמות לומדות תלויות זמניות ארוכות טווח על הרצף הדחוס. ה-Decoder מסוג U-Net מחזיר את הייצוג לאורך המקורי בעזרת skip connections, ומסווג כל נקודת זמן לאחד מ-5 מצבי מחזור הלב.

2. הסבר על סוגי השכבות השונים ברשת

שכבהסוגתפקיד
Conv1D Convolutional מחליקה פילטרים לאורך ציר הזמן וחולצת תבניות מקומיות (דופקים, שינויי אמפליטודה). הפילטר הראשון ברוחב 7 לוכד תבניות רחבות, והשני ברוחב 5 תבניות עדינות יותר.
BatchNormalization Normalization מנרמלת כל ערוץ לממוצע ≈0 וסטיית תקן ≈1 לאורך ה-Batch. מייצבת ומאיצה את תהליך האימון על ידי מניעת "סחיפת ערכים פנימית" (Internal Covariate Shift).
Dropout Regularization באופן אקראי מאפסת אחוז מהערכים בכל שלב אימון (10%–30%). מונעת מהרשת להסתמך על נוירונים בודדים ובכך מפחיתה Overfitting.
MaxPool1D Pooling מצמצמת את ציר הזמן פי 4 על ידי בחירת הערך המקסימלי מכל קבוצה של 4 נקודות. מקטינה מימדיות ומרחיבה את שדה הראייה (Receptive Field) של השכבות הבאות.
Bidirectional GRU Recurrent (BiRNN) שכבה רקורנטית דו-כיוונית מסוג GRU (Gated Recurrent Unit). סורקת את הרצף קדימה ואחורה ולומדת תלויות זמניות ארוכות טווח — למשל הקשר בין S1 ל-S2 במחזור דפיקת הלב. במודל זה מוערמות שתי שכבות BiGRU ב-Bottleneck להעמקת הייצוג הזמני. נבחרה GRU על פני LSTM בשל מספר פרמטרים נמוך יותר עם ביצועים דומים.
UpSampling1D Upsampling מגדילה בחזרה את ציר הזמן (כפולה של הפקטור). משמשת ב-Decoder של ה-U-Net כדי להחזיר את הייצוג לאורך המקורי.
Concatenate (Skip Connection) Merge משרשרת את הפלט של שכבת ה-Upsampling עם הפלט השמור מה-Encoder (skip connection). מאפשרת ל-Decoder לשחזר פרטים עדינים שאבדו בתהליך הדחיסה — עיקרון מרכזי בארכיטקטורת U-Net.
Dense Fully Connected שכבה צפופה (כל נוירון מחובר לכל הנוירונים בשכבה הקודמת). משמשת בסוף ה-Decoder להמרת הייצוג לסיווג סופי עבור כל נקודת זמן (5 מחלקות).
Softmax (activation) Activation ממירה את הפלט להתפלגות הסתברותית (סכום = 1). משמשת בראש הפלט כך שכל מחלקה מקבלת הסתברות, והמחלקה עם ההסתברות הגבוהה ביותר נבחרת כחיזוי.
ReLU (activation) Activation f(x) = max(0, x) — מאפסת ערכים שליליים. מוסיפה א-לינאריות לרשת, מה שמאפשר לה ללמוד תבניות מורכבות. ללא פונקציית הפעלה, מספר שכבות ליניאריות שקולות לשכבה אחת.

3. תיאור UML של המחלקות שנכתבו

בפרויקט נכתבה פונקציית שגיאה מותאמת אישית (custom loss function) ב-Python/Keras עבור שקלול מחלקות:

┌──────────────────────────────────────────────┐ │ weighted_loss (closure) │ ├──────────────────────────────────────────────┤ │ - weights_tensor: tf.Tensor │ │ (class weights from balanced calculation) │ ├──────────────────────────────────────────────┤ │ + loss_fn(y_true, y_pred) → weighted loss │ │ uses: tf.gather, sparse_categorical_CE │ └──────────────────────────────────────────────┘ ┌──────────────────────────────────────────────┐ │ _balanced_class_weights(counts) │ ├──────────────────────────────────────────────┤ │ weight(c) = total / (n_classes × count(c)) │ │ Background weight forced to 1.0 │ └──────────────────────────────────────────────┘

weighted_loss — פונקציית closure שמחזירה פונקציית Loss מותאמת. מקבלת טנזור משקלות מחלקות ומחזירה פונקציה loss_fn(y_true, y_pred) שמחשבת Sparse Categorical Cross-Entropy ומכפילה כל דגימה במשקל המחלקה שלה. כך מחלקות נדירות (כמו S1, S2) מקבלות עונש גבוה יותר.

_balanced_class_weights — פונקציה שמחשבת משקלות מאוזנים בסגנון sklearn: total / (n_classes × count). משקל מחלקת Background מקובע ל-1.0 כדי למנוע השפעה מוגזמת שלה.

המדד (metric) היחיד שהמודל עוקב אחריו הוא accuracy הסטנדרטי של Keras.

7. תיעוד והסבר של פונקציית השגיאה (Loss Function)

פונקציית השגיאה של המודל היא Weighted Sparse Categorical Cross-Entropy עבור הסגמנטציה:

Loss = weighted_seg_loss(y_true, y_pred)

7.1 Sparse Categorical Cross-Entropy (הבסיס)

שתי פונקציות השגיאה מבוססות על Sparse Categorical Cross-Entropy — פונקציית שגיאה סטנדרטית לסיווג רב-מחלקתי כאשר התוויות הן מספרים שלמים (ולא one-hot vectors). הנוסחה:

Loss(y, ŷ) = −log(ŷ[y])

כלומר, עבור כל דגימה, פונקציית השגיאה היא מינוס הלוגריתם של ההסתברות שהמודל הקצה למחלקה הנכונה. אם המודל בטוח בתשובה הנכונה (ŷ[y] קרוב ל-1), ה-Loss קרוב ל-0. אם המודל טועה (ŷ[y] קרוב ל-0), ה-Loss שואף לאינסוף — עונש כבד על טעויות בטוחות.

7.2 שקלול מחלקות (Class Weighting)

מכיוון שהדאטא-סט אינו מאוזן (למשל, מחלקת Diastole שולטת בסגמנטציה בעוד Background נדירה יחסית), שימוש ב-Cross-Entropy רגיל יגרום למודל "להתעצלן" ולחזות תמיד את המחלקה הנפוצה. לכן, כל דגימה מוכפלת במשקל ההפוך לשכיחות המחלקה שלה:

weight(class_c) = total_samples / (n_classes × count(class_c))

כך מחלקות נדירות (כמו Background, S2) מקבלות משקל גבוה יותר, ומחלקות נפוצות (כמו Diastole) מקבלות משקל נמוך — המודל "משלם" יותר על טעות במחלקה נדירה. משקל מחלקת Background מקובע ל-1.0. המשקלות שחושבו מהדאטא:

{Background: 1.0, S1: 1.015, Systolic: 0.894, S2: 1.161, Diastole: 0.523}

7.3 Loss לסגמנטציה (weighted_loss)

הפונקציה weighted_loss(weights_tensor) היא פונקציית closure — היא מקבלת את טנזור המשקלות כפרמטר ומחזירה פונקציה פנימית loss_fn(y_true, y_pred) שהיא פונקציית ה-Loss בפועל. מבנה זה נדרש מכיוון ש-Keras מצפה לפונקציית Loss עם חתימה קבועה של (y_true, y_pred) בלבד — ה-closure מאפשרת "ללכוד" את המשקלות בתוך הפונקציה מבלי להעבירם כפרמטר נוסף.

הפונקציה הפנימית loss_fn פועלת ברמת נקודת זמן — כל אחת מ-2,000 הנקודות בכל הקלטה מקבלת Loss בהתאם למחלקה האמיתית שלה (S1/S2/Systole/Diastole/Background) ומוכפלת במשקל המתאים. הפונקציה מחזירה טנזור בגודל (batch_size, 2000), ו-Keras מחשבת עליו ממוצע (reduce_mean) על פני כל נקודות הזמן וכל ההקלטות ב-Batch כדי לקבל את ה-Loss הסופי (סקלר).

8. תיעוד והסבר של ייעול ההתכנסות (Optimization)

8.1 אופטימייזר — Adam

המודל משתמש באופטימייזר Adam (Adaptive Moment Estimation). Adam משלב שני רעיונות: Momentum — שמחליק את כיוון הגרדיאנט לאורך זמן ומונע "זיגזוגים", ו-RMSProp — שמתאים את גודל הצעד לכל פרמטר בנפרד לפי גודל הגרדיאנטים האחרונים שלו. שילוב זה מאפשר התכנסות מהירה ויציבה יחסית לאופטימייזרים פשוטים יותר כמו SGD.

בנוסף, הופעל Gradient Clipping (clipnorm=1.0) — מנגנון שמגביל את הנורמה של וקטור הגרדיאנטים ל-1.0. אם הגרדיאנטים גדולים מדי (תופעת "התפוצצות גרדיאנטים" — Exploding Gradients), הם מנורמלים כדי למנוע צעדי עדכון קיצוניים שיכולים לערער את התכנסות המודל. תופעה זו שכיחה במיוחד ברשתות רקורנטיות (כגון ה-BiGRU בפרויקט זה).

8.2 התאמת קצב למידה — ReduceLROnPlateau

קצב הלמידה (Learning Rate) קובע את גודל הצעד בכל עדכון משקלות. קצב גבוה מדי עלול לגרום למודל "לקפוץ" מעל המינימום ולא להתכנס, ואילו קצב נמוך מדי יאט את ההתכנסות בצורה קיצונית.

בפרויקט זה נעשה שימוש ב-Callback של Keras בשם ReduceLROnPlateau, שמנטר את ה-val_loss ומפחית אוטומטית את קצב הלמידה כאשר אין שיפור. הפרמטרים:

  • patience=3 — אחרי 3 epochs ללא שיפור ב-val_loss, קצב הלמידה יורד.
  • factor=0.6 — קצב הלמידה מוכפל ב-0.6 (ירידה של 40%).
  • min_lr=1e-6 — גבול תחתון למניעת קצב זעיר מדי.

בפועל, קצב הלמידה ירד אוטומטית מ-1e-3 (התחלתי) ל-~7.8e-5 במהלך 30 epochs של אימון. הפחתה הדרגתית זו אפשרה למודל בתחילה לבצע צעדים גדולים לכיוון המינימום, ולאחר מכן צעדים קטנים ומדויקים יותר סביב נקודת האופטימום — מה שמשפר את ההתכנסות הסופית.

8.3 עצירה מוקדמת — EarlyStopping

מנגנון EarlyStopping מנטר את ה-val_loss ועוצר את האימון כאשר אין שיפור במשך 7 epochs רצופים (patience=7). כאשר העצירה מופעלת, המודל חוזר אוטומטית למשקלות של ה-epoch הטוב ביותר (restore_best_weights=True).

מנגנון זה משרת שתי מטרות: (1) חיסכון בזמן חישוב — אין טעם להמשיך אימון שלא משתפר; (2) מניעת Overfitting — ככל שהאימון נמשך, המודל עלול ללמוד רעש מדאטא האימון, מה שמשפר את שגיאת האימון אך מגדיל את שגיאת ה-Validation. העצירה המוקדמת תופסת את המודל ברגע האופטימלי — לפני שהוא מתחיל "לשנן" את הדאטא.

באימון המודל הסופי (model_13), מקסימום ה-epochs הוגדר ל-150, אך ה-EarlyStopping עצר את האימון אחרי 30 epochs בלבד. ה-val_loss הנמוך ביותר התקבל ב-epoch 22, ולאחריו 7 epochs ללא שיפור הפעילו את העצירה. המשקלות שנשמרו הם של epoch 22 — אותו מודל שהשיג val_loss=0.327.

9. תיעוד ההתמודדות עם הטיה ושונות (Bias & Variance)

9.1 מהי הטיה (Bias) ומהי שונות (Variance)?

הטיה (Bias) — שגיאה שנובעת מכך שהמודל פשוט מדי ולא מסוגל ללכוד את התבניות האמיתיות בנתונים. מודל עם הטיה גבוהה סובל מ-Underfitting — שגיאת האימון (Train Loss) עצמה גבוהה.

שונות (Variance) — שגיאה שנובעת מכך שהמודל רגיש מדי לדאטא האימון הספציפי ו"משנן" אותו במקום ללמוד את התבנית הכללית. מודל עם שונות גבוהה סובל מ-Overfitting — שגיאת האימון נמוכה, אך שגיאת ה-Validation/Test גבוהה בהרבה.

האיזון (Bias-Variance Tradeoff) — מודל אידיאלי מוצא איזון: מספיק מורכב כדי ללכוד את התבניות (הטיה נמוכה), אך לא מורכב מדי (שונות נמוכה). בפועל, המטרה היא למזער את הפער בין שגיאת האימון לשגיאת המבחן (Generalization Gap), תוך שמירה על שגיאת אימון נמוכה ככל שניתן.

9.3 מנגנונים שהופעלו לטיפול בשונות (Overfitting)

לאורך הפרויקט הופעלו מספר מנגנוני רגולריזציה כדי לשלוט בשונות:

מנגנוןכיצד מפחית שונותמימוש בפרויקט
Dropout באקראי מאפס נוירונים בכל צעד אימון, מה שמכריח את הרשת לפתח ייצוגים מבוזרים ולא להסתמך על נוירונים בודדים. שיעורים עולים בהדרגה: 10% (Encoder 1), 20% (Encoder 2), 20% (Decoder). Recurrent Dropout=20% ב-BiGRU.
BatchNormalization מנרמלת את הפלט בכל שכבה ומונעת סחיפת ערכים פנימית. פועלת גם כרגולריזציה קלה בגלל הרעש שהנורמליזציה לפי Mini-Batch מכניסה. אחרי כל שכבת Conv1D (Encoder ו-Decoder).
EarlyStopping עוצרת את האימון ברגע שה-val_loss מפסיק להשתפר, ומחזירה את המשקלות לנקודה הטובה ביותר — בדיוק לפני שה-Overfitting מתחיל להחמיר. patience=7, restore_best_weights=True. בפועל עצר ב-epoch 30 (best=22).
חלוקה לפי מטופלים (GroupShuffleSplit) מונעת Data Leakage — מוודאת שכל ההקלטות של מטופל נתון נמצאות רק בקבוצה אחת (Train/Val/Test). ללא זה, המודל יכול "לשנן" מאפיינים ייחודיים של מטופל ולהציג ביצועים אופטימיים מטעים. 70% Train / 15% Val / 15% Test, לפי GroupShuffleSplit עם random_state=42.

9.4 מנגנונים שהופעלו לטיפול בהטיה (Underfitting)

כדי למזער את ההטיה ולוודא שהמודל מסוגל ללכוד את מורכבות הדאטא:

מנגנוןכיצד מפחית הטיהמימוש בפרויקט
ארכיטקטורת U-Net עם BiGRU שילוב שכבות קונבולוציה (תבניות מקומיות) עם BiGRU (תלויות זמניות ארוכות טווח) ו-Skip Connections (שחזור פרטים עדינים) מספק כושר ביטוי (Capacity) גבוה. Encoder עם Conv1D, Bottleneck עם 2 שכבות BiGRU מוערמות, Decoder עם Skip Connections.
שתי שכבות BiGRU מוערמות העמקת הייצוג הזמני — השכבה הראשונה לומדת תלויות בסיסיות, והשנייה לומדת אינטראקציות מורכבות יותר ביניהן. שדרוג מ-model7 (BiGRU אחד) ל-model_13 (שני BiGRU מוערמים). תרם לשיפור משמעותי ב-Accuracy.
Class Weights מאוזנים מונע מהמודל "להתעצלן" ולחזות תמיד את המחלקה הנפוצה. נותן משקל גבוה יותר למחלקות נדירות (S1, S2) כך שהמודל לומד לזהות גם אותן. משקלות balanced: {BG:1.0, S1:1.015, Sys:0.894, S2:1.161, Dia:0.523}.

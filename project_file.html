שלב בניית ואימון המודל (Build and Train Deep Learning Model)

1. תיאור גרפי של ארכיטקטורת המודל

המודל הוא רשת עצבית עמוקה מסוג Conv1D-BiGRU בעלת שני ראשים (Dual-Head), המבוססת על ארכיטקטורת U-Net. הרשת מקבלת כקלט הקלטת PCG בת 2 שניות (2,000 נקודות זמן לאחר Downsampling) ומפיקה שני פלטים במקביל:

  • ראש סגמנטציה (Segmentation Head) — סיווג כל נקודת זמן לאחד מ-5 מצבים: S1, S2, Systole, Diastole, Background.
  • ראש זיהוי אוושה (Murmur Head) — סיווג ברמת ההקלטה כולה לאחת מ-3 קטגוריות: Present, Absent, Unknown.
Input: (2000, 1) │ ▼ ┌──────────────────────────────────────────────────┐ │ SHARED ENCODER (Encoder משותף) │ │ │ │ Conv1D(64, kernel=7) + BatchNorm + Dropout(0.1) │ │ → skip₁ (2000, 64) │ │ MaxPool1D(4) → (500, 64) │ │ │ │ Conv1D(128, kernel=5) + BatchNorm + Dropout(0.2)│ │ → skip₂ (500, 128) │ │ MaxPool1D(4) → (125, 128) │ └──────────┬───────────────────────┬───────────────┘ │ │ ▼ ▼ ┌─────────────────────┐ ┌─────────────────────────┐ │ Seg BiGRU │ │ Murmur BiGRU │ │ Bidirectional │ │ Bidirectional │ │ GRU(128) │ │ GRU(128) │ │ return_sequences │ │ return_sequences │ │ = True │ │ = False │ │ → (125, 256) │ │ → (256,) │ └────────┬────────────┘ └───────────┬─────────────┘ │ │ ▼ ▼ ┌────────────────────────┐ ┌──────────────────────┐ │ SEG DECODER (U-Net) │ │ MURMUR HEAD │ │ │ │ │ │ UpSample(4) → (500) │ │ Dense(64, relu) │ │ Concat(skip₂) │ │ Dropout(0.3) │ │ Conv1D(64, 5)+BN+DO │ │ Dense(3, softmax) │ │ │ │ → (3,) │ │ UpSample(4) → (2000) │ │ ↓ │ │ Concat(skip₁) │ │ Murmur class: │ │ Conv1D(5, 1, softmax)│ │ Present/Absent/ │ │ → (2000, 5) │ │ Unknown │ │ ↓ │ └──────────────────────┘ │ Per-timestep class: │ │ S1/S2/Sys/Dia/BG │ └────────────────────────┘

כפי שניתן לראות, ה-Encoder המשותף מעבד את הסיגנל ומפיק ייצוג קומפקטי. מנקודה זו הרשת מתפצלת — כל ראש מקבל שכבת BiGRU עצמאית המתמחה במשימה שלו. ראש הסגמנטציה משתמש ב-Decoder מסוג U-Net אשר מחזיר את הייצוג לאורך המקורי בעזרת skip connections, בעוד ראש האוושה מרכז את הייצוג לווקטור בודד ומסווג ברמת ההקלטה.

2. הסבר על סוגי השכבות השונים ברשת

שכבהסוגתפקיד
Conv1D Convolutional מחליקה פילטרים לאורך ציר הזמן וחולצת תבניות מקומיות (דופקים, שינויי אמפליטודה). הפילטר הראשון ברוחב 7 לוכד תבניות רחבות, והשני ברוחב 5 תבניות עדינות יותר.
BatchNormalization Normalization מנרמלת כל ערוץ לממוצע ≈0 וסטיית תקן ≈1 לאורך ה-Batch. מייצבת ומאיצה את תהליך האימון על ידי מניעת "סחיפת ערכים פנימית" (Internal Covariate Shift).
Dropout Regularization באופן אקראי מאפסת אחוז מהערכים בכל שלב אימון (10%–30%). מונעת מהרשת להסתמך על נוירונים בודדים ובכך מפחיתה Overfitting.
MaxPool1D Pooling מצמצמת את ציר הזמן פי 4 על ידי בחירת הערך המקסימלי מכל קבוצה של 4 נקודות. מקטינה מימדיות ומרחיבה את שדה הראייה (Receptive Field) של השכבות הבאות.
Bidirectional GRU Recurrent (BiRNN) שכבה רקורנטית דו-כיוונית מסוג GRU (Gated Recurrent Unit). סורקת את הרצף קדימה ואחורה ולומדת תלויות זמניות ארוכות טווח — למשל הקשר בין S1 ל-S2 במחזור דפיקת הלב. נבחרה GRU על פני LSTM בשל מספר פרמטרים נמוך יותר עם ביצועים דומים.
UpSampling1D Upsampling מגדילה בחזרה את ציר הזמן (כפולה של הפקטור). משמשת ב-Decoder של ה-U-Net כדי להחזיר את הייצוג לאורך המקורי.
Concatenate (Skip Connection) Merge משרשרת את הפלט של שכבת ה-Upsampling עם הפלט השמור מה-Encoder (skip connection). מאפשרת ל-Decoder לשחזר פרטים עדינים שאבדו בתהליך הדחיסה — עיקרון מרכזי בארכיטקטורת U-Net.
Dense Fully Connected שכבה צפופה (כל נוירון מחובר לכל הנוירונים בשכבה הקודמת). משמשת בראש האוושה להמרת הווקטור המסכם לסיווג סופי.
Softmax (activation) Activation ממירה את הפלט להתפלגות הסתברותית (סכום = 1). משמשת בשני ראשי הפלט כך שכל מחלקה מקבלת הסתברות, והמחלקה עם ההסתברות הגבוהה ביותר נבחרת כחיזוי.
ReLU (activation) Activation f(x) = max(0, x) — מאפסת ערכים שליליים. מוסיפה א-לינאריות לרשת, מה שמאפשר לה ללמוד תבניות מורכבות. ללא פונקציית הפעלה, מספר שכבות ליניאריות שקולות לשכבה אחת.

3. תיאור UML של המחלקות שנכתבו

בפרויקט נכתבו מספר מחלקות מותאמות אישית (custom classes) ב-Python/Keras עבור מדדי הערכה ופונקציות שגיאה:

┌─────────────────────────────────────────────────┐ │ keras.metrics.Metric (base class) │ └────────────────────┬────────────────────────────┘ │ inherits ┌────────────┴────────────┐ ▼ ▼ ┌──────────────────┐ ┌────────────────────┐ │ SparseRecall │ │ SparsePrecision │ ├──────────────────┤ ├────────────────────┤ │ - class_id: int │ │ - class_id: int │ │ - tp: Variable │ │ - tp: Variable │ │ - fn: Variable │ │ - fp: Variable │ ├──────────────────┤ ├────────────────────┤ │ + update_state() │ │ + update_state() │ │ + result() │ │ + result() │ │ + reset_state() │ │ + reset_state() │ │ + get_config() │ │ + get_config() │ └──────────────────┘ └────────────────────┘ Recall = TP/(TP+FN) Precision = TP/(TP+FP)

SparseRecall — מטריקה מותאמת אישית המחשבת Recall (רגישות) עבור מחלקה ספציפית. בכל batch היא סופרת True Positives (חיזויים נכונים עבור המחלקה) ו-False Negatives (מקרים שייכים למחלקה אך לא זוהו). הנוסחה: Recall = TP / (TP + FN). מחלקה זו נדרשת כיוון ש-Keras לא תומך באופן מובנה ב-Recall עבור sparse labels עם פלט softmax רב-מחלקתי.

SparsePrecision — מטריקה מותאמת אישית המחשבת Precision (דיוק) עבור מחלקה ספציפית. סופרת True Positives ו-False Positives (דגימות שהמודל חזה כמחלקה זו אך בפועל שייכות למחלקה אחרת). הנוסחה: Precision = TP / (TP + FP).

שתי המחלקות יורשות מ-keras.metrics.Metric ומממשות את הממשק הנדרש: update_state לעדכון המונים בכל batch, result לחישוב הערך הסופי, ו-reset_state לאיפוס בתחילת כל epoch.

בנוסף, נכתבו שתי פונקציות שגיאה מותאמות: weighted_seg_loss ו-weighted_murmur_loss (ראו פירוט בסעיף פונקציית השגיאה בהמשך).

7. תיעוד והסבר של פונקציית השגיאה (Loss Function)

פונקציית השגיאה הכוללת של המודל מורכבת משתי פונקציות שגיאה — אחת לכל ראש — המשוקללות יחד:

Total Loss = SEG_LOSS_WEIGHT × Seg_Loss + MURMUR_LOSS_WEIGHT × Murmur_Loss
Total Loss = 1.0 × Seg_Loss + 0.1 × Murmur_Loss

7.1 Sparse Categorical Cross-Entropy (הבסיס)

שתי פונקציות השגיאה מבוססות על Sparse Categorical Cross-Entropy — פונקציית שגיאה סטנדרטית לסיווג רב-מחלקתי כאשר התוויות הן מספרים שלמים (ולא one-hot vectors). הנוסחה:

Loss(y, ŷ) = −log(ŷ[y])

כלומר, עבור כל דגימה, פונקציית השגיאה היא מינוס הלוגריתם של ההסתברות שהמודל הקצה למחלקה הנכונה. אם המודל בטוח בתשובה הנכונה (ŷ[y] קרוב ל-1), ה-Loss קרוב ל-0. אם המודל טועה (ŷ[y] קרוב ל-0), ה-Loss שואף לאינסוף — עונש כבד על טעויות בטוחות.

7.2 שקלול מחלקות (Class Weighting)

מכיוון שהדאטא-סט אינו מאוזן (למשל, מחלקת Background/Unrecognized שולטת בסגמנטציה, ומחלקת Absent שולטת באוושה), שימוש ב-Cross-Entropy רגיל יגרום למודל "להתעצלן" ולחזות תמיד את המחלקה הנפוצה. לכן, כל דגימה מוכפלת במשקל ההפוך לשכיחות המחלקה שלה:

weight(class_c) = total_samples / (n_classes × count(class_c))

כך מחלקות נדירות (כמו Present באוושה) מקבלות משקל גבוה יותר, ומחלקות נפוצות (כמו Absent) מקבלות משקל נמוך — המודל "משלם" יותר על טעות במחלקה נדירה. הממוצע המשוקלל שומר על סקאלת ה-Loss זהה.

7.3 Loss לראש הסגמנטציה (weighted_seg_loss)

פועלת ברמת נקודת זמן — כל אחת מ-2,000 הנקודות בכל הקלטה מקבלת Loss בהתאם למחלקה האמיתית שלה (S1/S2/Systole/Diastole/Background) ומוכפלת במשקל המתאים. ה-Loss הסופי הוא הממוצע על פני כל נקודות הזמן בכל ההקלטות ב-Batch.

7.4 Loss לראש האוושה (weighted_murmur_loss)

פועלת ברמת הקלטה — כל הקלטה מקבלת Loss בהתאם לתווית האמיתית שלה (Present/Absent/Unknown) ומוכפלת במשקל המתאים.

7.5 שקלול בין הראשים

ה-Loss הכולל משוקלל כך שמשקל ראש הסגמנטציה (1.0) גבוה פי 10 ממשקל ראש האוושה (0.1). הסיבה: הסגמנטציה היא המשימה העיקרית והמורכבת יותר (סיווג 2,000 נקודות), ואילו זיהוי האוושה הוא סיווג ברמת הקלטה. ללא שקלול זה, Loss של האוושה עלול "לחטוף" את כיוון הגרדיאנטים ולפגוע באיכות הסגמנטציה. משקל זה נקבע אמפירית — ב-model9 הורד מ-0.5 ל-0.1 ונצפה שיפור בביצועי הסגמנטציה.
